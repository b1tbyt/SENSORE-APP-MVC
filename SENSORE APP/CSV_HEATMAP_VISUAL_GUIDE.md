## ?? API ENDPOINTS

### Endpoint 1: Get Available Heatmaps

```
REQUEST:
GET /Patient/GetAvailableHeatmaps

RESPONSE:
{
  "heatmaps": [
    "1cfd777_20251011.csv",
    "1cfd777_20251012.csv",
    "71e6bab3_20251011.csv",
    ...
  ],
  "count": 12
}

USAGE:
List all available CSV files
```

### Endpoint 2: Load Specific Heatmap

```
REQUEST:
POST /Patient/LoadHeatmapFromCsv?fileName=1cfd777_20251011.csv

RESPONSE:
HTTP 302 Redirect to /Patient/Index
TempData: "Heatmap 'filename' loaded successfully"

USAGE:
Load specific CSV file and display
```

### Endpoint 3: Load Random Heatmap

```
REQUEST:
POST /Patient/LoadHeatmapFromCsv

RESPONSE:
HTTP 302 Redirect to /Patient/Index
TempData: "Random heatmap loaded from CSV data"

USAGE:
Load random CSV file
```

### Endpoint 4: Management Info

```
REQUEST:
GET /Patient/HeatmapManagement

RESPONSE:
{
  "availableHeatmaps": [...],
  "totalCount": 12,
  "message": "12 heatmap(s) available"
}

USAGE:
Get management information
```

## ?? PERFORMANCE

### Memory Usage

```
Per Heatmap:
?? byte[32,32] = 1,024 bytes
?? Loading CSV: ~2KB temporary
?? Total per file: ~1KB resident

Total System:
?? 12 CSV files: ~12 KB
?? Current heatmap: 1 KB
?? Target heatmap: 1 KB
?? Total overhead: ~15-20 KB
```

### CPU Usage

```
Per Update Cycle (100ms):
?? Read random number: < 0.1 ms
?? Load CSV (50% of time): ~1-5 ms
?? Interpolate values: ~2-3 ms
?? Notify subscribers: < 0.5 ms
?? Total: ~5-10 ms per cycle

Result: Minimal impact on system
```

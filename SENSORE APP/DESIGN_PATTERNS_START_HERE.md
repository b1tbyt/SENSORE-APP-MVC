## ?? THE THREE PATTERNS

### ?? Factory Pattern
- **Type**: Creational
- **Problem**: Object creation complexity
- **Solution**: Centralized factory interface
- **File**: `Services/Factories/MessageFactory.cs`
- **Benefit**: Encapsulation and validation

### ?? Adapter Pattern
- **Type**: Structural
- **Problem**: Incompatible interfaces
- **Solution**: Adapter converts between formats
- **File**: `Services/Adapters/PressureSensorAdapter.cs`
- **Benefit**: Legacy system integration

### ?? Strategy Pattern
- **Type**: Behavioral
- **Problem**: Multiple algorithms for same task
- **Solution**: Interchangeable algorithm objects
- **File**: `Services/Strategies/RiskAnalysisStrategy.cs`
- **Benefit**: Runtime algorithm switching

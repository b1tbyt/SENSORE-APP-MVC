# Live Heatmap System Architecture

## Components

### 1. **PressureDataService** (Services/PressureDataService.cs)
- **Role**: Simulates real-time pressure sensor data
- **Updates**: Every 100ms
- **Process**:
  - Maintains `_currentHeatmap` (live data)
  - Maintains `_targetHeatmap` (simulated sensor input)
  - Generates random hotspots when condition met (30% chance)
  - Smoothly interpolates (25% per frame) from current ? target
  - Fires `OnHeatmapUpdated` event after each update

### 2. **HeatmapBroadcastService** (Services/HeatmapBroadcastService.cs)
- **Role**: Bridges PressureDataService to SignalR
- **Process**:
  - Subscribes to `OnHeatmapUpdated` event
  - Converts 2D byte array ? jagged byte array
  - Broadcasts to all connected clients via `PressureHub`

### 3. **PressureHub** (Hubs/PressureHub.cs)
- **Role**: SignalR WebSocket endpoint
- **Route**: `/pressureHub`
- **Messages**:
  - `HeatmapUpdated` - Sends updated heatmap data to clients

### 4. **PatientController** (Controllers/PatientController.cs)
- **Role**: Serves the dashboard view
- **Action**: `Index(string timeframe)`
- **Data**: Gets initial heatmap from `PressureDataService`

### 5. **JavaScript Client** (Views/Patient/Index.cshtml)
- **Role**: Receives updates and renders heatmap
- **Connection**: SignalR WebSocket to `/pressureHub`
- **Handler**: Listens for `HeatmapUpdated` messages
- **Rendering**: Updates table cell background colors

## Key Improvements Made

? **Separated Concerns**
- PressureDataService: Data generation only
- HeatmapBroadcastService: Broadcasting logic
- Controller: Request handling only

? **Singleton Pattern**
- PressureDataService registered as Singleton
- Single background task generating data
- All clients receive the same data stream

? **Performance Optimizations**
- Color caching: Avoid recalculating RGB values
- Efficient jagged array conversion
- 100ms update interval: Smooth without overwhelming network

? **Robust Connection Handling**
- Auto-reconnect with exponential backoff
- Error logging in browser console
- Graceful handling of missing DOM elements

? **Real-time Simulation**
- Hotspots appear, move, and fade naturally
- Smooth interpolation between states
- 30% chance per frame to generate new target state
